## ğŸ”¹ Procedural Programming (Left side)

**Idea:**
Focuses on **functions/procedures** that act on data.

**What the picture shows:**

* Data (like files, money, records) is **separate**
* Functions like **withdraw, deposit, transfer** keep moving between data
* Data is passed around from one function to another

**Key points:**

* Data is **not protected**
* Any function can change the data
* Code becomes hard to manage as programs grow

**Example (Java-style idea):**

```java
deposit(account, amount);
withdraw(account, amount);
```

---

## ğŸ”¹ Object-Oriented Programming (Right side)

**Idea:**
Focuses on **objects** that contain **both data and methods**.

**What the picture shows:**

* A **Customer object** holds money
* Actions (deposit, withdraw) are done **inside the object**
* Data and behavior are **bound together**

**Key points:**

* Data is **encapsulated** (more secure)
* Easy to maintain and modify
* Models real-world entities (customer, bank, account)

**Example (Java-style idea):**

```java
customer.deposit(amount);
customer.withdraw(amount);
```
